What is PGN?

PGN stands for Portal Game Notation. It's a plain text format for recording and sharing entire chess games in a human-readable and machine-parsable way.

Why Use PGN?

PGN is simple, standardized, and supports both metadata and move history. It's widely supported by chess engines, databases, GUIs, and web tools. It allows easy storage, review analysis, and transmission of complete games.

Basic Structure of a PGN File

A PGN file has two main parts:

  1. Tag pairs (metadata)
  2. Movetext (the actual game)

Example PGN 

[Event "Casual Game"]
[Site "Local Club"]
[Date "2023.01.15"]
[Round "?"]
[White "Alice"]
[Black "Bob"]
[Result "1-0"]

1. e4 e5 2. Nf3 Nc6 3. Bb5 a6 4. Ba4 Nf6 5. O-O Be7 6. Re1 b5
7. Bb3 d6 8. c3 O-O 9. h3 Nb8 10. d4 Nbd7 11. Nbd2 Bb7 1-0

Explanation

  • Tag pairs use square brackets and contain key details about the game (players, location, result, etc.).
  • Movetext lists the moves using Standard Algebraic Notation. Numbers indicate turn count. Game ends with the result: 1-0 (White wins), 0-1 (Black wins), or 1/2-1/2 (draw).

Summary

PGN is the de facto format for full-game representation in chess. It's human-readable, editable in any text editor, and supported across nearly all chess tools (including this one). Ideal for saving, sharing, and studying games.
